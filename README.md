# плэйн

Платформа для визуального вдохновения и творчества. Креаторы создают мудборды с произвольным расположением изображений, пользователи находят их через поиск и навигацию по двумерному эмоциональному спектру, а также взаимодействуют с контентом через лайки и комментарии.

## Архитектура

### Роли пользователей

**Admin** — управляет заявками на креаторство, может блокировать и удалять аккаунты авторов.

**Creator** — одобренный пользователь с личным кабинетом и редактором мудбордов. Может ставить лайки и оставлять комментарии к чужим работам.

**User** — самостоятельно зарегистрированный пользователь. Может просматривать опубликованные мудборды, ставить лайки и оставлять комментарии.

Единая точка входа `/api/creators/login` — проверяет сначала таблицу `admins`, затем `creators`, возвращает JWT с полем `role`. Пользователи (User) входят через отдельный эндпоинт `/api/users/login`.

### Система заявок

```
Заявка (pending) → Одобрение админом → Токен активации (7 дней)
→ Пользователь устанавливает пароль → Аккаунт Creator активирован
```

Ссылка активации генерируется на бэкенде и передаётся вручную — без email-сервиса.

### Редактор мудбордов

Drag-and-drop холст 1200×800px. Каждый элемент хранит `x`, `y`, `width`, `height`, `url`. Layout сериализуется в JSON и сохраняется в поле `layout` таблицы `frames`. При просмотре координаты восстанавливаются точно — пользователь видит расположение как задумал автор.

```json
{
  "items": [
    { "id": "1", "url": "/uploads/photo.jpg", "x": 50, "y": 80, "width": 300, "height": 400 }
  ]
}
```

### Двумерный эмоциональный спектр

Каждый мудборд имеет координаты `mood_x` и `mood_y` (значения от 0 до 1), задаваемые автором при создании или редактировании работы:

- ось X: от минимализма (0) к максимализму (1)
- ось Y: от тёмной тональности (0) к светлой (1)

Эндпоинт `GET /api/mood-spectrum` возвращает все опубликованные мудборды с их координатами. Мудборды без заданных координат отображаются в центре плоскости со значениями (0.5, 0.5). На клиенте спектр визуализируется через WebGL (библиотека OGL): каждая точка кликабельна и ведёт на страницу мудборда.

### Социальное взаимодействие

Лайки и комментарии доступны как пользователям (User), так и авторам (Creator). Повторный лайк от одного участника исключён на уровне базы данных через ограничение уникальности. История лайков доступна в профиле пользователя и в личном кабинете автора.

### Поиск

`GET /api/frames?search=query` — ищет одновременно по названию мудборда, описанию, тегам, имени и юзернейму креатора через `OR` в одном SQL-запросе с `ilike`. Тот же фильтр поддерживается эндпоинтом эмоционального спектра.

## API

### Публичные
| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/frames` | Список мудбордов (поиск по `search`, `tag`) |
| GET | `/api/frames/:id` | Мудборд по ID |
| GET | `/api/tags` | Все теги |
| GET | `/api/profiles/:username` | Публичный профиль креатора |
| GET | `/api/profiles/search?q=` | Поиск креаторов |
| GET | `/api/mood-spectrum` | Мудборды с координатами эмоционального спектра |

### Пользователи `/api/users`
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/register` | Регистрация пользователя |
| POST | `/login` | Вход пользователя |
| GET | `/me` | Профиль текущего пользователя |
| GET | `/me/likes` | История лайков пользователя |
| POST | `/frames/:id/like` | Поставить / убрать лайк (user и creator) |
| GET | `/frames/:id/likes` | Количество лайков у мудборда |
| GET | `/frames/:id/my-like` | Проверить, лайкнул ли текущий пользователь |
| GET | `/frames/:id/comments` | Список комментариев к мудборду |
| POST | `/frames/:id/comments` | Добавить комментарий (user и creator) |
| DELETE | `/frames/:id/comments/:comment_id` | Удалить свой комментарий |
| GET | `/creator-likes` | История лайков автора |

### Креаторы `/api/creators`
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/login` | Единый вход (admin + creator) |
| POST | `/applications` | Подать заявку на креаторство |
| POST | `/set-password` | Установить пароль по токену активации |
| GET | `/me` | Профиль текущего креатора |
| GET | `/me/frames` | Мои мудборды |
| POST | `/me/frames` | Создать мудборд |
| PUT | `/me/frames/:id` | Обновить мудборд (layout, теги, публикация, mood_x, mood_y) |
| DELETE | `/me/frames/:id` | Удалить мудборд |
| PATCH | `/me/frames/:id/toggle` | Переключить статус публикации |
| POST | `/me/upload` | Загрузить изображение |
| GET | `/boards/:id` | Публичный просмотр мудборда с данными автора и спектра |

### Админ `/api/admin`
| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/login` | Вход администратора |
| GET | `/creators/applications` | Список заявок |
| POST | `/creators/applications/:id/approve` | Одобрить, сгенерировать ссылку активации |
| POST | `/creators/applications/:id/reject` | Отклонить заявку |
| GET | `/creators` | Все аккаунты креаторов |
| DELETE | `/creators/:id` | Удалить аккаунт и все мудборды |
| PATCH | `/creators/:id/toggle` | Заблокировать / разблокировать |
| POST | `/upload` | Загрузить изображение |

## Запуск

### Бэкенд
```bash
cd server
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Создать .env
echo "DATABASE_URL=postgresql://user:password@localhost:5432/moodboard" > .env
echo "SECRET_KEY=your-secret-key" >> .env

# Создать таблицы и первого администратора
python create_tables.py
python create_admin.py

# Применить миграции (новые сущности)
python migrate_social.py
python migrate_mood_axes.py

uvicorn app.main:app --reload
```

### Фронтенд
```bash
cd client
npm install
npm run dev
```

### Проверка типов
```bash
cd client
npm run typecheck
```

## База данных

```
admins               — аккаунты администраторов
creators             — аккаунты одобренных креаторов
users                — аккаунты самостоятельно зарегистрированных пользователей
creator_applications — заявки (pending / approved / rejected)
frames               — мудборды (layout как JSON, mood_x и mood_y для спектра)
tags                 — теги
frame_tag            — связь многие-ко-многим между мудбордами и тегами
likes                — лайки (от user или creator к frame)
comments             — комментарии (от user или creator к frame)
```
